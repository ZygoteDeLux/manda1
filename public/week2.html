Understand scoping in Javascript
and know to always use const if you can get away with it otherwise let.

Scope refers to the context in which variables are declared, accessed, and manipulated. There are three types of scope:
Global Scope:
Variables declared outside any function or block are globally scoped.
Function Scope:
Variables declared within a function are function scoped.
Block Scope:
Variables declared within a block
{}
are block scoped.

Var, Let, and Const:

var: Function-scoped, can be redeclared and reassigned.
let: Block-scoped, can be reassigned but not redeclared.
const: Block-scoped, cannot be reassigned or redeclared.

Why Prefer const over let?

Immutability: Using const signifies that a variable's value will not be reassigned, making code easier to reason about.
Error Prevention: Accidental reassignment is less likely.
Readability: Indicates intention that a variable should remain constant.

When to Use let?

Use let when you know that the value of the variable will change over time.

Understand advanced things about functions in Javascript.
Can write callback functions with different syntax.

First-Class Functions: In JavaScript, functions are first-class objects, meaning they can be assigned to variables, passed as arguments, and returned from other functions.
Higher-Order Functions: Functions that take another function as an argument or return a function as a result.
Closures: Functions have access to variables from outer scopes even after the outer function has returned.
Arrow Functions: A shorthand syntax for writing function expressions.
Async/Await: Modern way to handle asynchronous operations, built on top of Promises.

Understands how to send data with GET requests over HTTP and knows how to do it in Express.

in HTTP, you can send data with GET requests by appending query parameters to the URL.
Query parameters are key-value pairs that come after the question mark "?" in the URL. Multiple query parameters are separated by the ampersand "&" character.
http://example.com/users?name=John&age=30
In this URL, name and age are query parameters with the values John and 30 respectively.

In an Express app, you can access these query parameters using the req.query object in your route handler function. Here's an example:

In an Express app, you can access these query parameters using the req.query object in your route handler function. Here's an example:
import express from 'express';

const app = express();

// Define a route to handle GET requests
app.get('/users', (req, res) => {
// Extract query parameters from the request
const { name, age } = req.query;

// Do something with the data
res.send(`Received name: ${name}, age: ${age}`);
});
// Start the server on port 3000
app.listen(3000, () => {
console.log('Server is running on http://localhost:3000');
});
