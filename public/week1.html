Can work with Git in the terminal.

use the the following command to initialize a new repository:
git init
now link to a remote repository you want to push your changes to.
git remote add origin [URL]
after initializing git, you stage all changes to git typing the command:
git add -A
you can now commit the staged files with the command:
git commit -m "commit message here"
The -u flag sets the upstream, linking your local branch to the corresponding remote branch.
git push -u origin [branch_name]
After this initial setup, you can use
git push
to push future commits to the same remote branch.

Type coercion and conversion

Type coercion in JavaScript refers to the automatic or implicit conversion of values from one data type to another.
This occurs during the evaluation of expressions involving mixed types. JavaScript tries to predict what you mean and performs a type coercion to accomplish that.
let x = "5";
let y = 10;
// Type coercion: the number 10 is implicitly converted to a string
let z = x + y; // z becomes a string "510", not 15
Type conversion, on the other hand, refers to the manual or explicit conversion between types. This is done using built-in functions.
let a = "5";
let b = 3;
// Type conversion: we explicitly convert string to number
let c = parseInt(a) + b; // c becomes 8, not "53"

//parseInt(): Converts to integer
//parseFloat(): Converts to floating-point number
//String(): Converts to string
//Boolean(): Converts to boolean

Can explain what Node.js is and how to run files with it.

Node.js is an open source cross platform runtime environment built on Chrome's V8 JavaScript engine that allows you to execute JavaScript code server-side.
Unlike client-side JavaScript that runs in a browser, Node.js enables backend development, API creation, and interaction with databases, among other tasks.
After installing node.js one way you can execute a JavaScript file is by navigating to its location in the terminal and use the command:
node filename.js

Can design a REST API and knows the two REST conventions we will follow and why they exist and what they map to.

Standardization of the endpoints which should reflect the underlying collections in your system.

Collection Endpoints (e.g., /users):
GET: Retrieve all users from the 'users' collection.
POST: Create a new user within the 'users' collection.
Single Resource Endpoints (e.g., /users/{id}):
GET: Retrieve a specific user by ID.
PUT: Update a specific user by ID.
DELETE: Delete a specific user by ID.

Ordering of the HTTP methods.

The hierarchy we practice in our rest API is GET >> POST >> PATCH >> PUT >> DELETE
