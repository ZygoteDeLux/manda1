Week 3

Understands the use case for Nodemon and its limitation for development.

Nodemon watches your Node.js application files and automatically restarts the server whenever changes are detected.
This saves time during development by eliminating the need to manually stop and restart the Node.js application.

Nodemon is intended for development purposes only. It is not recommended for production environments. It consumes more system resources since it keeps watching files for changes, which might be an issue for very large projects.

Can use loop methods (+map, +filter, %reduce). Can choose the right one at the appropriate time.
Understands why a functional approach to loops is to be preferred and how to implement it without side effects.

There are mainly 2 types of loops in javascript
  • for loops
  • while loops
for loop is used when you know how many times you want to run a loop.
while loop is used when you dont know how many times you want to run a loop until a condition is met.

Functional approach.

Map
const arr = [1, 2, 3];
const doubled = arr.map(x => x * 2);

Immutability: A functional approach often avoids changing state or data.
This makes the code easier to understand and debug.
alternatively the same operation could be done using a for loop.
let arr = [1, 2, 3];
let doubled = [];

for (let i = 0; i < arr.length; i++) {

 doubled.push(arr[i] * 2);
}


State Modification: The doubled array is declared outside of the loop and its state is modified within the loop.
This makes it harder to reason about the state of doubled at any point in the program.

Filter
const arr = [1, 2, 3, 4];

const evenNumbers = arr.filter(x => x % 2 === 0);
Reduce
const items = [

 { name: "chips", price: 20 },

 { name: "chicken", price: 100 },

 { name: "pork", price: 200 },
];

const totalPrice = items.reduce((total, item) =>{

 return total + item.price;

}, 0);

Body parsing is used for a number of reasons, one of them is for data sensitivity.
Query strings are visible in the URL, making them inappropriate for confidential information. Request bodies are more secure.

Can use the right terminology when it comes to the different parts of an URL.

The URL (Uniform resource locator) has ten parts.
The protocol, subdomain, top-level domain, second-level domain, subdirectory, parameter, port,
path, query, and fragment. A URL doesn't have to contain all of these parts at once
I will go over the protocol, domain-name, query and path.
eg.

https://www.animals.com/goats?id=7
    "https://" is the protocol.
    "www" is the subdomain.
    "animals" is the second-level domain.
    "com" is the top level domain.
    "goats" is the path.
    "id=7" is the query string
